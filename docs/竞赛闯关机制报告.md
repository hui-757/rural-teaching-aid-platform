# 竞赛闯关过关机制报告

## 一、整体架构

竞赛闯关采用**西游记地图**作为入口，共 **9 个地界**（对应 9 种计算题分类），每个地界下设 **10 个关卡**。用户选择地界后进入该分类的闯关流程。

### 地界与分类映射

| 地界 | 分类 | 说明 |
|------|------|------|
| 花果山 | 口算乘法 | 基础乘法口算 |
| 高老庄 | 不进位笔算乘法 | 简单竖式乘法 |
| 流沙河 | 连续进位笔算乘法 | 进阶竖式乘法 |
| 白虎岭 | 中间有0的乘法 | 含0乘数竖式 |
| 盘丝洞 | 末尾有0的乘法 | 末尾含0竖式 |
| 火焰山 | 积的变化规律 | 规律题 |
| 通天河 | 乘法估算与数学文化 | 估算+文化题 |
| 乌鸡国 | 口算除法 | 除法口算 |
| 大雷音寺 | 笔算除法竖式 | 除法竖式 |

## 二、关卡配置体系

10 个关卡的难度通过 `LEVEL_CONFIG` 数组配置，各参数含义如下：

```typescript
const LEVEL_CONFIG = [
  { timeLimit: 60, targetAccuracy: 0.15,  consecutiveRequired: 1, questionCount: 3 },  // 第1关
  { timeLimit: 60, targetAccuracy: 0.175, consecutiveRequired: 1, questionCount: 3 },  // 第2关
  { timeLimit: 60, targetAccuracy: 0.175, consecutiveRequired: 1, questionCount: 3 },  // 第3关
  { timeLimit: 60, targetAccuracy: 0.2,   consecutiveRequired: 2, questionCount: 3 },  // 第4关
  { timeLimit: 60, targetAccuracy: 0.2,   consecutiveRequired: 2, questionCount: 4 },  // 第5关
  { timeLimit: 60, targetAccuracy: 0.225, consecutiveRequired: 2, questionCount: 4 },  // 第6关
  { timeLimit: 60, targetAccuracy: 0.225, consecutiveRequired: 2, questionCount: 4 },  // 第7关
  { timeLimit: 60, targetAccuracy: 0.25,  consecutiveRequired: 2, questionCount: 4 },  // 第8关
  { timeLimit: 60, targetAccuracy: 0.25,  consecutiveRequired: 3, questionCount: 5 },  // 第9关
  { timeLimit: 60, targetAccuracy: 0.275, consecutiveRequired: 3, questionCount: 5 },  // 第10关
]
```

### 参数说明

| 参数 | 含义 | 变化趋势 |
|------|------|----------|
| `timeLimit` | 每关限时（秒） | 固定 60 秒 |
| `targetAccuracy` | 目标正确率阈值 | 从 15% 递增至 27.5% |
| `consecutiveRequired` | 要求连续正确题数 | 前期 1 题，中期 2 题，后期 3 题 |
| `questionCount` | 本关最少答题数 | 前期 3 题，中期 4 题，后期 5 题 |

## 三、过关判定逻辑（`checkPass` 函数）

```typescript
const checkPass = (answers: AnswerItem[], config: typeof LEVEL_CONFIG[0]) => {
  // 1. 答题数量门槛：必须达到本关要求的最少答题数
  if (answers.length < config.questionCount) return false

  // 2. 正确率计算：答对数 / 总答题数
  const correctCount = answers.filter(a => a.correct).length
  const accuracy = correctCount / answers.length

  // 3. 最大连续正确数计算
  let maxConsecutive = 0
  let currentConsecutive = 0
  for (const a of answers) {
    if (a.correct) {
      currentConsecutive++
      maxConsecutive = Math.max(maxConsecutive, currentConsecutive)
    } else {
      currentConsecutive = 0
    }
  }

  // 4. 双条件判定：正确率达标 AND 连续正确数达标
  return accuracy >= config.targetAccuracy && maxConsecutive >= config.consecutiveRequired
}
```

### 判定条件拆解

| 条件 | 说明 | 示例（第5关） |
|------|------|---------------|
| 答题数量 | 必须答够 `questionCount` 题 | 至少答 4 题 |
| 正确率 | 答对数 / 总答题数 ≥ `targetAccuracy` | 正确率 ≥ 20% |
| 连续正确 | 历史答题中最大连续正确数 ≥ `consecutiveRequired` | 至少连续对 2 题 |

**关键特性**：过关判定是**实时进行的**——每答一题后立即检查，只要任一时刻满足条件即刻过关，无需答完所有题目。

## 四、闯关流程时序

### 1. 选择地界 → 进入分类
- 用户点击地图上的地界（如"高老庄"）
- 系统设置 `selectedCategory = '不进位笔算乘法'`
- 调用 `loadQuestions()` 从数据库获取该分类全部题目并随机洗牌

### 2. 选择关卡 → 开始闯关
- 第 1 关默认解锁
- 后续关卡需要**前一关通关**才能解锁（`isAvailable = level === 1 || catLevels.includes(level - 1)`）
- 点击关卡后弹出**学生标识输入框**（`studentName`），要求输入学号/标识
- 确认后进入答题界面，计时器开始 60 秒倒计时

### 3. 答题过程
- 题目从随机洗牌后的数组中按顺序取出（`currentQIndex % questions.length`）
- 支持三种题型：
  - **普通口算题**：直接输入答案
  - **乘法竖式**：交互式 blank 输入，中间过程显示 □ 占位符，只输入最终结果
  - **除法竖式**：需输入商和余数
- 提交后即时判定对错，用颜色条显示答题历史（绿色=对，红色=错）

### 4. 闯关结束触发条件

闯关结束由以下任一条件触发：

| 触发条件 | 处理逻辑 |
|----------|----------|
| **满足过关条件** | `checkPass()` 返回 true，调用 `finishLevel()` |
| **时间耗尽** | `timeLeft <= 0`，调用 `finishLevel()` |
| **题目循环耗尽** | 实际上会循环取题（`% questions.length`），理论上不会耗尽 |

### 5. 结果结算与保存（`finishLevel`）

```typescript
const finishLevel = async (finalAnswers: AnswerItem[]) => {
  // 1. 判定是否通过
  const passed = checkPass(finalAnswers, config)
  setIsPass(passed)
  setShowResult(true)
  setIsPlaying(false)

  // 2. 若通过，更新本地通关状态
  if (passed && selectedCategory) {
    setCompletedLevels((prev) => ({
      ...prev,
      [selectedCategory]: [...new Set([...prev[selectedCategory], gameState.currentLevel])]
    }))
  }

  // 3. 自动保存到数据库 record 表
  if (user && selectedCategory && currentGrade) {
    const score = Math.round((correctCount / finalAnswers.length) * 100)
    await supabase.from('record').insert({
      teacher_id: user.id,
      student_name: studentName || '匿名',
      category: selectedCategory,
      grade: currentGrade,
      level: gameState.currentLevel,
      score,
      answers_json: finalAnswers,
    })
    fetchRankings(selectedCategory)  // 刷新排行榜
  }
}
```

### 4. 结果弹窗
- 显示"闯关成功"或"闯关失败"
- 展示答对/总题数、用时、各题正误详情
- 提供教师备注输入框（可选）
- 点击"返回关卡"或"确认"关闭弹窗

## 五、解锁机制

```typescript
const isCompleted = catLevels.includes(level)           // 本关是否已通过
const isAvailable = level === 1 || catLevels.includes(level - 1)  // 是否解锁
```

- **第 1 关**：始终解锁
- **第 N 关（N > 1）**：必须第 N-1 关已通关（`catLevels.includes(N - 1)`）
- **已通关关卡**：显示皇冠图标，可重复挑战
- **未解锁关卡**：显示锁图标，不可点击

## 六、题目随机性

```typescript
const loadQuestions = async (category: string) => {
  const { data } = await supabase
    .from('calc_question')
    .select('*')
    .eq('category', category)
    .eq('grade', grade)

  if (data) {
    const shuffled = [...data].sort(() => Math.random() - 0.5)
    setQuestions(shuffled as CalcQuestion[])
  }
}
```

- **每次进入关卡**都会重新从数据库获取题目并随机洗牌
- 题目通过 `currentQIndex % questions.length` 循环取用
- 若题库题目数少于答题数，会出现**循环复用**

## 七、数据持久化

### 自动保存（record 表）

| 字段 | 说明 |
|------|------|
| `teacher_id` | 教师用户 ID |
| `student_name` | 学生标识（闯关时输入） |
| `category` | 计算题分类 |
| `grade` | 年级 |
| `level` | 关卡序号（1-10） |
| `score` | 得分（正确率百分比） |
| `answers_json` | 答题明细（题目ID、学生答案、正误） |
| `created_at` | 自动记录时间 |

### 通关进度持久化

- 通关状态从 `record` 表反查：
  ```typescript
  const passedLevels = data
    .filter((r) => r.score >= config.targetAccuracy * 100)
    .map((r) => r.level)
  ```
- 页面加载时自动读取历史通关记录，恢复解锁状态

## 八、排行榜机制

```typescript
const fetchRankings = async (category: string) => {
  const { data } = await supabase
    .from('record')
    .select('*')
    .eq('category', category)
    .eq('grade', currentGrade)
    .order('score', { ascending: false })
    .limit(20)
}
```

- 按 `score` 降序排列，取前 20 条
- 展示学生标识、关卡、分数
- 每次闯关结束后自动刷新

## 九、机制总结图

```
用户点击地界
    ↓
选择关卡（第1关默认解锁，后续需前置通关）
    ↓
输入学生标识 → 开始闯关（60秒倒计时）
    ↓
逐题答题（随机题库，即时判定）
    ↓
每答一题 → checkPass() 检查过关条件
    ↓
    ├─ 满足条件 ─→ 闯关成功 ─→ 自动保存记录 ─→ 解锁下一关
    └─ 时间耗尽 ─→ 闯关失败 ─→ 自动保存记录 ─→ 保持原状态
```
